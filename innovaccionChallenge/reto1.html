Conceptos básicos

Azure

Servicio de nube de Microsoft que entrega de servicios informaticos a través de internet.

Cómputo en la nube

Servicios de computo entregados por internet.

SLA (Service Level Agreement)

Disponibilidad y rendimiento en porcentaje, que tendrán los recursos durante un tiempo determinado.

Alta disponibilidad

Servicio continuo sin inactividad aún existiendo fallas, dependiendo del SLA.

Tolerancia a fallos

Existen respaldos de información, y geodistribución en la nube para que en caso de alguna falla y los datos se mantengan seguros.

Agilidad

Rápidez de la implementación y configuración de los recursos en la nube.

Elasticidad

Autoescalación de recursos a medida de la necesidad actual. Siempre se cuenta con los recursos que se necesitan.

Escalabilidad horizontal

Incremento de la capacidad de cómputo mediante más instancias de la misma.

Escalabilidad vertical

Incremento de la capacidad de cómputo agregando más RAM o CPU al recurso existente.

CapEx

Gasto inicial de dinero en Infraestructura física.

OpEx

Dinero usado para seguir operando.

Nube pública

Los servicios se ofrecen a través de internet y están disponibles para cualquiera que desee adquirirlos, de este modo se obtienen servicios de nube de terceros. Por ejemplo, los servicios que ofrece Microsoft Azure.

Nube privada

Recursos informáticos o información de una propiedad privada.

Nube híbrida

Entorno que combina una nube pública con una nube privada, lo cual permite compartir datos y aplicaciones entre ellas.

Modelos de servicio

PaaS (Platform as a Service)

Modelo de servicio en un entorno de alojamiento gestionado donde el proveedor se encarga de las maquinas virtuales y recursos de red, mientras que el inquilino solamente de las aplicaciones.

SaaS (Software as a Service)

Servicio donde el proveedor de la nube brinda las aplicaciones y se encarga de ellas, mientras que el inquilino solo aporta y se encarga de los datos.

IaaS (Infraestructure as a Service)

Modelo de servicio donde el proveedor sólo se encarga del hardware rentado, mientras que el cliente se encarga del software.

Economías de escala

Recursos

Elemento administrable y disponible en Azure.

Grupo de recursos

Contenedor lógico de incluye los recursos relacionados a una solución.

Regiones

Áreas geográficas del planeta que tienen por lo menos un centro de datos de Azure.

Geografías

Zona que contiene al menos 2 regiones de Azure.

Zonas de disponibilidad

Centros de datos separados físicamente dentro de una región de Azure, con electricidad, refrigeración y equipo independientes. Esto srive para tener un centro de datos de respaldo.

Servicios Azure

Compute

Servicios de cómputo o procesamiento bajo demanda, tales como las maquinas virtuales o kubernetes.

Network

Servicios de red para conectar recursos con el mundo exterior, el mejor ejemplo es Azure Virtual Network.

Storage

Servicios de almacenamiento.

Database

Servicios de base de datos para diferentes volumenes.

Big data

Servicios para el procesamiento y análisis de una gran cantidad de información o registros.

IoT (Internet de las cosas)

Servicios para conectar y recibir información de diferentes dispositivos electrónicos.

Inteligencia artificial

Servicios de aprendizaje automático.

DevOps

Permite que los roles que antes estaban aislados (desarrollo, operaciones de TI, ingeniería de la calidad y seguridad) se coordinen y colaboren para crear productos mejores y más confiables.
